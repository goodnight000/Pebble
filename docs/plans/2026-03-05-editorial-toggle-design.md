# Dual Feed View Design

**Goal:** Add a toggleable editorial/newsroom presentation without removing the existing wireframe view.

**Approved Direction:** Preserve the current industrial wireframe interface as one mode and add a second, cleaner editorial mode optimized for reading clarity, hierarchy, and scanability.

**Scope:**
- Add a persistent in-app view toggle for `Wireframe` and `Editorial`
- Restyle digest panel, breaking alert, and feed cards in editorial mode
- Keep the same backend data and feed behavior
- Do not remove or regress the existing wireframe mode

**Editorial Principles:**
- Lead with title and source, not decorative chrome
- Use sentence case for editorial headlines/cards
- Reduce shadows, dashed borders, and all-caps density
- Promote source, trust, and timestamp into a readable metadata rail
- Tighten summaries and card height for faster scanning
- Keep strong visual personality through typography and layout, but make readability dominant
